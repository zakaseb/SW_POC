# Halcon JAMA-compliant MBSE Requirement Intelligence Platform
# Python: 3.12 (validated on 3.12.13)
#
# Single-file install for a plain venv (no conda):
#
#     python3.12 -m venv .venv && source .venv/bin/activate
#     python -m pip install --upgrade pip
#     # 1) install the PyTorch stack for YOUR hardware (see block below)
#     # 2) install everything else:
#     pip install -r requirements_sw_final.txt
#
# -----------------------------------------------------------------------------
# IMPORTANT - read before installing
# -----------------------------------------------------------------------------
# The reference environment runs on an NVIDIA DGX Spark (GB10 Grace-Blackwell,
# sm_121, aarch64, CUDA 13) using PyTorch *nightly* cu130 wheels:
#       torch==2.14.0.dev20260622+cu130
#       torchvision==0.29.0.dev20260623+cu130
#       torchaudio==2.11.0.dev20260623+cu130
#       triton==3.7.1
# These wheels are NOT on PyPI, are aarch64/CUDA-13 specific, and rotate off
# the nightly index over time. Pinning them here is exactly why a plain
# `pip install` fails on a different machine. So torch is intentionally NOT
# pinned in this file - install it FIRST, picking the wheel index for your box:
#
#   CPU only (any machine):
#       pip install torch torchvision torchaudio
#
#   x86_64 + CUDA 12 GPU:
#       pip install torch torchvision torchaudio \
#         --index-url https://download.pytorch.org/whl/cu124
#
#   DGX Spark / Blackwell (aarch64, CUDA 13) - matches the reference env:
#       pip install --pre torch torchvision torchaudio \
#         --index-url https://download.pytorch.org/whl/nightly/cu130
#
# The nvidia-* CUDA runtime libraries are pulled in automatically as
# dependencies of the torch wheels, so they are deliberately not listed here.
#
# NOTE: the reference conda env also contained tensorflow, keras, nipype,
# nibabel and friends. None of those are imported by this project, so they are
# left out on purpose. The obsolete `pathlib` 1.0.1 backport that was also in
# that env is omitted as well - it breaks pathlib on Python 3 if installed.
# =============================================================================
